Overview

This is a simplified guide on the basics of how to use common C++ compilers from the command line.

How Long Does This Take?

This guide should probably require about 30 minutes to read in total, or even less if you only read the sections on GCC or MSVC.

The goal is for this to be short and to the point, an efficient and worthwhile investment of your time.

With that in mind, it will probably require more time to fully comprehend and try the examples yourself, but ideally this guide can be completed in less than an hour.

Before You Begin

Be sure to read the Command Line Primer

Motivations

Whether you use an IDE or not, your C++ compiler is a command line program.

If you use an IDE, then your IDE typically just runs this command line program on your behalf, hopefully in a correct way with a successful outcome, however this is not always guaranteed.

Whether you use an IDE or not, it is useful to learn how to compile programs on the command line so that you can better troubleshoot problems and make better use of your IDE and other tools.

You will also be better able to understand common and important build systems like CMake, and you will be better able to do things like use other libraries in your programs.

Many of the configuration steps you may need to perform in your IDE or build system simply mirror those same options accepted by the command line compiler, so learning this will help you use other tools.

Windows Setup

Install Visual Studio.

  • It’s FREE! and easy to install.
  • It does everything you want.
  • It comes with a command line compiler.

WARNING

Visual Studio is required to follow along with this guide for Windows.

This guide does not cover other compilers for Windows.

Devtools PowerShell

See: How To Open The Developer PowerShell

If you are using Windows, do not use the regular PowerShell be sure to open the special Devtools PowerShell instead.

STOP

Use the Developer PowerShell NOT the Developer Command Prompt. The Developer Command Prompt is old and bad. The examples in this guide are written for Developer PowerShell only.

Using PowerShell 7

You really should use PowerShell 7 and not the built in default PowerShell, however Microsoft Visual Studio only provides launchers for the the default old bad PowerShell 5 out of the box. You will need to configure PowerShell 7 to run the VS devtools.

UNIX Setup

Linux

You may need to install GCC, including the g++ command line compiler first.

Use your package manager such as apt or yum to install what you need.

For example see: How To Install GCC On Ubuntu

Mac

Install XCode and the XCode Command Line Tools.

INFO

You don’t need the XCode IDE for this guide because this guide is about the command line, however you will need the XCode Command Line Tools.

Install Mac Homebrew. Homebrew requires the XCode Command Line Tools.

WARNING

On MacOS, you will use Clang instead of GCC, so any time you see the g++ command, run clang++ instead, otherwise Clang and GCC are mostly the same.
