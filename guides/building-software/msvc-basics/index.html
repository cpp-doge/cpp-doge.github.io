About The Compiler

The Microsoft command line compiler program is named cl.exe and the command line linker program is named link.exe.

INFO

We can omit the .exe when running these, so to run the compiler we simply run the cl command or to run the linker we simply run the link command.

Using Visual Studio

Here’s how to use cl on the command line to build and run a simple C++ program consisting of a single main.cpp source file.

C:\> cd project
C:\project> cl -EHsc main.cpp
C:\project> .\main
Hello World!

WARNING

You need the -EHsc flag or you will probably get some warnings relating to exceptions.

WARNING

The .\ is important, you must use a fully qualified path to run your program.

Change The Name

Change the program name by adding the -Fe flag.

C:\> cd project
C:\project> cl -EHsc main.cpp -Fe"hello.exe"
C:\project> .\hello
Hello World!

The -Fe flag stands for Executable File.

Multiple Files

If your program consists of multiple .cpp files you can simply list them all when you compile. The commands are otherwise the same as before. This works best when all files are in the same folder.

C:\> cd project
C:\project> cl -EHsc main.cpp foo.cpp -Fe"hello.exe"
C:\project> .\hello
Hello World!

Discrete Builds

Building and running a C++ program actually consists of multiple steps that are hidden from you. Your compiler can handle all of these steps automatically but you can have more power and flexibility if you learn how to perform these individual steps yourself, by hand. It is also just generally good to be aware of and know how this works.

Overview

  • Step 1: Compile individual .cpp source files into Object Files
  • Step 2: Link multiple Object Files into your final program

Step 1 Compile

Compile each individual .cpp file into a corresponding object file by using the -c flag for MSVC. This means compile only.

cl -EHsc -c foo.cpp -Fo"foo.obj"
cl -EHsc -c main.cpp -Fo"main.obj"

The -Fo flag is similar to the -Fe flag, it gives a name to the resulting object file.

INFO

The .o means Object File for GCC but .obj means object file for MSVC.

Step 2 Link

Link all your object files into the final resulting program.

link foo.obj main.obj -out:hello.exe

Notice that there are different commands for compiling versus linking.

Notice that the flag to control output file name is also different for the linker versus the compiler.

Step 3 Run

Now you can run your program as before.

.\hello
Hello World!
