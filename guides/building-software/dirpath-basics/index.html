Directory Paths

When you compile a program your compiler will search in a large number of different directories for things like header include files and link library files.

Depending on your OS and file structure, you may need to observe some other considerations around the files and folders of your project including your programs and libraries.

Concepts Overview

  • Include File Search Directories
  • Library File Search Directories
  • Binary Search Directories
  • Library Loader Directories

File Search Directories

Most C++ programs will use so called header include files or just headers for short.

You typically see a lot of this kind of thing in most C++ code.

#include "my_program.hpp"
#include "my_wrappers.hpp"
#include "my_utilities.hpp"

#include <iostream>
#include <string>
#include <vector>

You will inevitably end up needing a way to tell the compiler about other directories that contain headers,

If the compiler cannot find your header includes you will get compile errors.

This is also true of libraries. When you link a library to your program by name, the compiler searches a set of directories for library files.

If the compiler cannot find your library files you will get compile errors.

Binary Directories

When running a program that uses shared libraries, your program needs to know how to find those libraries.

Linux

The LD_LIBRARY_PATH variable has already been described for GCC, but there are other ways including the use of -rpath.

More on this later.

Windows

When working with DLLs on windows, all your DLLs need to be findable by your program. A simple way to ensure your program has what it needs is to make sure you always put your .dll files in the same directory as your .exe files.

However, this may be confounded based several factors. It is a common need to configure your build to copy your .dll files to the same directory as your .exe files.

More on this later.
