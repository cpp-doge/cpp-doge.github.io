Files & Folders

Long ago, we used to store information on bits of dead trees on our desks and in boxes. No more.

Photo by Maksym Kaharlytskyi

You have files on your computer. You probably have things like photos, music, videos, spreadsheets, video game saves and many other possible kinds of files.

Your files are organized in folders, much like a traditional old school filing cabinet. One major difference is that folders on a computer can have many other folders and files inside them where usually a physical filing cabinet does not typically have folders inside of other folders (but it could, who knows!?)

Folders on a computer are more technically known as “directories”.

The Current Working Directory

Any program you run has a current working directory.

Think of this as the folder where the program believes it lives.

This allows programs to, for example, use files they might need that are part of the program like fonts, sounds, photos or videos.

The current directory can be changed at any time, either before you run a program, or the program might change its own current directory as it runs.

Your shell also has a current working directory and it will usually be shown so you know where you are.

When you run a program from your shell then the program you run will inherit the same current working directory as your shell.

File Paths

You need a way to get to the files and folders on your computer.

A file path is a special unique way to know exactly how to get to any specific file or folder.

A file path is usually a list of folder names separate with slashes. Linux uses the forward slash / while Windows uses the backslash \.

Some example file paths on Linux might look like this.

/usr/local/include
/home/elmo/MyFile.txt

Some example file paths on Windows might look like this.

C:\Windows\System32
C:\Users\elmo\MyFile.txt

Linux is case sensitive, Windows is not.

Paths & Spaces

Having spaces in your file or directory names can cause a lot of pain.

Avoid it if you can.

Name your files with other characters like _, - or .

Sometimes you can't avoid it like with C:\Program Files.

For example, instead of

My Very Scientific Research Paper.doc

substitute underscores for spaces

My_Very_Scientific_Research_Paper.doc

or use dashes

My-Very-Scientific-Research-Paper.doc

or using dots is also fine, but maybe a little weird

My.Very.Scientific.Research.Paper.doc

File Extensions

How do you know what’s in a file? How do you know if a file is a video or an MS Word document?

A simple answer is file extensions.

A file extension is really only a simple matter how you name your files. When you create or rename a file, you can put a period . followed by short abbreviated name describing what is in the file.

For example

  • miami_vacay37.jpg means a JPEG photo
  • to_do_list.txt means a plain text file
  • so_much_words.doc means an MS Word document

These are known as MIME Types.

You are not required to have extensions in your file names and you are not required to follow MIME Types.

What's more, file extensions may not match their file contents!

This information helps decide what to do with the file, like if you want all images to open in Photoshop and you want all movies to open in Final Cut.

You may receive a warning when renaming a file, but this is often ok.

If you created a new text file on accident when you wanted to create an HTML file then renaming the new file from .txt to .html is probably fine.

If you are trying to rename summer_camp.jpg to My Essay.doc then that won’t work.

Windows hides file extensions by default. You should probably disable this setting in View Options.

Filesystem Root

Your files are stored on your hard drive using a special kind of computer program called a Filesystem.

Filesystems aren’t really separate programs, they are part of your operating system.

Files in a filesystem are organized in a tree like structure. Folders are like the branches of the tree and files are like the leaves.

Every filesystem has a root folder. This is like the trunk or roots of the tree. There is only one trunk, but there are many branches. This is the starting place for all the other files on your computer.

The Real Root

On Linux your root folder is simply named /.

On Windows your root folders have a drive name.

The default windows root folder is named C:\ because C: is the default system drive name and \ means root folder sort of similar to Linux, but it’s a backslash on Windows because that’s what Windows uses.

Path Types

There are two kinds of paths, absolute paths and relative paths.

Absolute Paths

An absolute path is sometimes called a full path.

An absolute path gives the full and exact location of a file or folder.

Absolute paths must begin at the root folder and name all the nested subfolders that your file is in.

For eaxmple this is an absolute path.

C\Users\elmo\documents\drawings\pokemon\2022\September\my_art.png

As you can see, absolute paths may become long and cumbersome.

Relative Paths

A relative path gives the path of a file or folder relative to the current working directory.

There is a special relative path named .. that means the parent folder.

Let’s pretend our current working directory is

C:\Users\elmo\blog\posts

then this relative path

..\..\images\doge.png

refers to the file

C:\Users\elmo\images\doge.png

Current Relative Directory

There is another special path named . that just means the current working directory. This may not seem useful but it has its uses, for example, you may need to use this if you want to run a script.

Let’s pretend our current working directory is

C:\Users\elmo\blog\posts

then this relative path

.\wow-much-doge.html

refers to the file

C:\Users\elmo\blog\posts\wow-much-doge.html

Fully Qualified Paths

Some things need fully qualified paths to work correctly.

A fully qualified path is either an absolute path or a relative path that begins with the . or .. special directory names.

This is a relative path but it is NOT a fully qualified path

hello/world.txt

This IS a fully qualified path

./hello/world.txt
