Scripting

You can write programs using the programming language of your shell, usually called your shell scripting language.

These programs can do almost anything.

Some ideas for things you could do

  • Backup your vacation photos
  • Backup your game saves
  • Run different Minecraft versions with different mods
  • Launch a video game with a high priority for better performance
  • Compile and run a program you wrote
  • Compile and run a program someone else wrote
  • Manage a Git repository
  • Rename hundreds of files at once
  • Resize, rotate, crop and blur hundreds of images at once

Aliases

Aliases are a way to make common commands shorter, easier and faster to type and run.

Stretching List

Let’s pretend you want ls to always show the long detailed version of your list of files.

You can create an alias for this in Bash with

alias ls="ls -l"

Now when you run the ls command you will get a long detailed list of files.

Making Remove Nicer

Deleting a directory in PowerShell can be a bit cumbersome

Remove-Item -Recurse -Force .\some_folder_i_dont_want

We can make this shorter with an alias like this

Set-Alias -Name "mydel" -Value Remove-Item -Recurse -Force

And now we can simply run

mydel .\some_folder_i_dont_want

Easy CD To Project

Let’s pretend you have a C++ project living in

/home/elmo/projects/cpp/all/games/myengine

Typing out this path can be cumbersome, so make an alias

alias myengine="cd /home/elmo/projects/cpp/all/games/myengine"

Now you can run the myengine command to easily and quickly change into your awesome game engine project directory.

PowerShell

You can similarly create aliases with PowerShell, for example our previous alias might look like this in powershell.

Set-Alias -Name "myengine" -Value Set-Location C:\home\elmo\projects\cpp\all\games\myengine

Variables

You can set variables in your shell like you do in other programming languages.

Bash

my_var="Hello"

echo "$my_var"

PowerShell

$MyVar="Hello"

Write-Output "$MyVar"

The System Path

Be sure to read

  • About Environment Variables

What Is It

The system path is a list of directories where your shell will look for programs when you try to run commands.

This list of directories is stored in the PATH environment variable on Windows, Mac and Linux.

If you run a command without a fully qualified path then your shell will search through each directory in the system path list to see if there are any programs with that name.

If a program name is found in one of those directories then the shell runs the program as you would run any other command from the command line.

This is one of the main ways that you are able to run many shell commands by only their name, without needing to give a qualified path.

Path Separators

Environment variables are just strings, so the system path is just a string.

There needs to be a way to pack a list of many directories into a single string.

The system path separator is a single character that separates each path in the list.

Linux uses the colon : as the system path separator.

Windows uses semicolon ; as the system path separator.

Take A Look

Have a look at your current system PATH for yourself

Bash

echo "$PATH"

PowerShell

Write-Output "$env:PATH"

OK!

Your system path may naturally be long and confusing to look at.

Using This To Your Advantage

Adding System Paths

You can use the system path to your advantage. You can write your own programs and put them in one of the system path directories so that your commands are easily and readily available on the command line.

Aliases are convenient, but when you want to do more complicated things you will probably want to write standalone shell script programs to accomplish your goals.

Run Your Own Scripts

It is very convenient to create a new directory where you will keep the shell scripts that you write.

Add this directory to your system path and suddenly you have the power to write infinitely many infinitely flexible commands to do whatever you want quickly and with ease.

For example

Bash

cd /home/elmo

# Create a new directory for your scripts
mkdir scripts

# Create a new script file
touch scripts/dostuff

# Edit your script to do whatever you want
nano scripts/dostuff

# Add your new scripts directory to the system path
# so that you can run your new commands directly
export PATH="/home/elmo/scripts:$PATH"

# On linux you will need to make your script executable
chmod 755 scripts/dostuff

# Now you can run your script like you
# would run any other normal command
dostuff

PowerShell

Set-Location C:\Users\elmo

# Create a new directory for your scripts
New-Item -Path scripts -ItemType Directory

# Create a new script file
New-Item -Path scripts\dostuff.ps1 -ItemType File

# Edit your script to do whatever you want
notepad scripts\dostuff.ps1

# Add your new scripts directory to the system path
# so that you can run your new commands directly
$env:PATH = "C:\Users\elmo\scripts;$env:PATH"

# Now you can run your script like you
# would run any other normal command
dostuff
