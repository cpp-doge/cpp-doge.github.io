Basic Boolean Tests

You can use basic boolean tests. This mostly works based on whether another command succeeds or not.

Bash

The Test command is the special character [.

The Test command on Linux lets you check if something is true or not.

The test command contains what should hopefully be mostly familiar expressions and ends with ], like this

[ "hello" == "world" ]

This checks to see if hello is the same as world.

STOP

Be sure to leave space after [ and before ].

PowerShell

PowerShell test commands are performed with various special operators like -eq meaning equal.

"hello" -eq "world"

If Statements

You can check if something is true and then do something or not, based on the result.

Bash

In Bash you can use the test [ command from earlier with if statements.

if [ "hello" == "world" ]; then
  echo "Yes they are equal"
else
  echo "NO they are equal"
fi

STOP

The semi-colon ; after ] and before then is important, don’t forget!

PowerShell

PowerShell if statements have a C like syntax.

if ("hello" -eq "world") {
  Write-Output "Yes they are equal"
} else {
  Write-Output "NO they are equal"
}

Checking For Success

You can check to see if a command was successful or not.

The Exit Code

The exit code of a program is a plain integer number that the program gives back to the shell when the program is finished running.

An exit code of zero 0 means success.

All other exit codes of mean failure or error.

You should return reasonable error codes from the programs you write if you detect an error.

The meaning of the non zero error codes you use when writing your own programs is your free choice.

Exit Code Examples

Bash

In bash, you can simply write command instead of using the [ test command. The test command itself works by setting the exit code.

if /usr/local/bin/mycommand; then
  echo "Success"
else
  echo "FAIL"
fi

PowerShell

In PowerShell you can check the last exit code from the $LASTEXITCODE variable.

C:\my\command.exe

if ($LASTEXITCODE -eq 0) {
  Write-Output "Success"
} else {
  Write-Output "FAIL"
}

More File Stuff

You can check to see if a file or folder exists or not.

Checking For Files

Bash

Check if a file exists, -f means file.

if [ -f "./myfile.txt" ]; then
  echo "Found myfile"
else
  echo "myfile NOT FOUND"
fi

Check if a folder exists, -d means directory.

if [ -d "./myfolder/" ]; then
  echo "Found myfolder"
else
  echo "myfolder NOT FOUND"
fi

PowerShell

Check if a file exists

if (Test-Path -Path ".\myfile.txt" -Type Leaf) {
  echo "Found myfile"
} else {
  echo "myfile NOT FOUND"
}

Check if a folder exists

if (Test-Path -Path ".\myfolder\" -Type Container) {
  Write-Output "Found myfolder"
} else {
  Write-Output "myfolder NOT FOUND"
}

Checking If Files Do NOT Exist

This works the same for both directories and files.

Bash

To check if a file does NOT exist, just add the not ! operator to your logic.

if [ ! -f "./myfile.txt" ]; then
  echo "Found myfile"
else
  echo "myfile NOT FOUND"
fi

PowerShell

To check if a file does NOT exist, just add the -not operator to your logic.

if (-not (Test-Path -Path ".\myfile.txt" -Type Leaf)) {
  echo "myfile NOT FOUND"
} else {
  echo "Found myfile"
}

Pipes & Redirection

You can send the output of a command into a other commands. This is known as using pipes. A pipe is like a connection from one program to another through which data can flow.

You can also send the output of a command into a file. You might want to do this, for example if you have a command that produces a lot of output and you want to save the result so you can look at it later.

This is common when dealing with programming related things like compiling, building, installing and testing programs. It can also be useful if you have something like a game server and you want to save a history log of who connects to your game and what they say and do.

This can be useful for things like admins moderating a game server and other similar situations. This can also be useful for troubleshooting your scripts and programs.

Writing Output To Files

You can send the output of a command into a file instead. You can do this with the greater than > symobl. This is called redirection.

Bash

echo "Hello this is going into the file" > myfile.txt

PowerShell

Write-Output "Hello this is going into the file" > myfile.txt

WARNING

This will normally output UTF-16 when using the default Windows PowerShell 5.

This will likely cause problems with Linux like tools such as GCC, Perl, Grep Sed and Awk.

Use PowerShell 7 for better interoperation with Linux tools.

Another Glorified Calculator

You can connect the output of one command to the input of another. You can do this with the vertical bar | symbol, also known as the pipe symbol. This is called piping.

Bash

The bc command on Linux allows you to do math. It can run files like scripts, or you can send it some input with a pipe.

For example in Bash, you can send the output of the echo command into the bc command like this.

echo "2 + 2" | bc

PowerShell

This works mostly the same in PowerShell but you don’t need to use bc to do basic math because you can just do math directly in PowerShell.

Using Tee

You may have noticed in the output to file example that you no longer see the output in your terminal.

You can fix this by using the tee command. The tee command takes input from a pipe and sends it into a file but also still prints it to your terminal so you can see what’s going on.

Bash

echo "Hello this is going into the file" | tee myfile.txt

PowerShell

In PowerShell, the command is called Tee-Object.

Write-Output "Hello this is going into the file" | Tee-Object myfile.txt

For Each File

You can loop over lists of files and do something with each one.

Bash

for f in ./myfolder/*.mp3; do echo "The File Is $f"; done

You can also use find exec

find ./myfolder -iname "*.mp3" -exec "The File Is {}"\;

WARNING

The \; may seem odd, but it’s important. This is too much to cover here, see the find command documentation for more info.

PowerShell

$files = Get-ChildItem -Filter *.mp3 .\myfolder

foreach ($f in $files) { Write-Output "The File Is: $f" }

You can also pipe command output to for each.

Get-ChildItem -Filter *.mp3 .\myfolder | foreach { Write-Output "The File Is: $_" }

Compound Commands

Sometimes you may want more power than a simple command provides, but you don’t want to write a full separate script.

You can run complex compound commands by running a sub shell and giving it a command to run.

Sometimes using a sub shell is necessary.

Bash

bash -c 'echo "Hello from the sub shell"'

PowerShell

pwsh -Command { Write-Output "Hello from the sub shell" }
