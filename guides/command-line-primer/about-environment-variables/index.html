Environment Variables

An environment variable is a way to save some info in a place where your programs can always get to it.

You can set environment variables so that all programs have them or only for one program at a time.

Syntax

Environment variables look like regular variables in your shell scripting language.

Bash environment variables begin with $ and the name, so for example $HELLO could be an environment variable in Bash.

PowerShell environment variables begin with $env: and the name, so for example $env:HELLO could be an environment variable in PowerShell.

Common Variables

There are some common environment variables that will already be set on your system.

On Linux the $HOME environment variable points to your local user foler. On Windows the $env:USERPROFILE environment variable points to your local user folder.

Try printing these from the command line and see what you get.

with Bash

echo "My User Dir Is $HOME"

with PowerShell

Write-Host "My User Dir Is $env:HOME"

These variables are useful because they help you write scripts that run about the same on any computer, even when different people have different usernames. Maybe those different people are not different people but you just have two computers and the username is different between them.

Setting Environment Variables

Set environment variables in Bash like you would set a normal scripting variable, but add the word export like this.

export MYVAR="hello"

Set environment variables in PowerShell like you would set a normal scripting variable, but add the env: prefix like this.

$env:MYVAR="hello"

Temporarily

You can set an environment variable when you run a program and it will only apply to that program as it runs.

In Bash you can do this by simply putting a plain variable assignment directly in front of your command.

SOMEVAR="hello" ./someprogram

In PowerShell you can do this by running a command in a subshell with the Command option.

pwsh -Command { $env:SOMEVAR="hello"; Write-Host "Some Var Is $env:SOMEVAR" }
