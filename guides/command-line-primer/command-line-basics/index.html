The Command Prompt

When you open your terminal, you will be greeted with your shell prompt.

Bash

In Bash it looks something like this

elmo@doge:~$

The little squiggle ~ is special, it is short for your local user directory.

PowerShell

In PowerShell it looks something like this

C:\Users\elmo>

The prompt usually shows you some information about your computer and where you are, like your username and the path to the current working directory.

Running Commands

You run a command on the command line by simply typing out the command and hitting the Enter key.

Bash

Running a command in Bash looks something like this

elmo@doge:~$ firefox

PowerShell

Running a command in PowerShell looks something like this

C:\> explorer.exe

Echo

You can print messages with echo or Write-Output, doing so looks something like this.

Bash

elmo@doge:~$ echo "hello this is my message"
hello this is my message
elmo@doge:~$

PowerShell

C:\Users\elmo> Write-Output "hello this is my message"
hello this is my message

C:\Users\elmo>

From now on, examples may not show the full command prompt, but only the commands to be run.

Many Linux commands can also be run in PowerShell for example the echo command can be used in PowerShell too

Escaping

Some characters are special because what you’re actually doing is writing tiny short snippets in a shell scripting language and then running them as commands.

If you want to print a literal double quote, for example, then you will need to escape it.

Bash

Escape double quotes with backslashes in Bash

echo "the message is \"hello\""
the message is "hello"

PowerShell

Escape double quotes with the backtick in PowerShell

Write-Output "the message is `"hello`""
the message is "hello"

Doing Basic Math

You can do basic arithmetic directly from your shell.

PowerShell

In PowerShell you can simply type it out and hit enter

C:\Users\elmo> 2 + 2
4
C:\Users\elmo>

Bash

In Bash you can use the bc calculator program.

elmo@doge:~$ echo "2 + 2" | bc
4
elmo@doge:~$

Don’t worry too much about how this works for now.

See Advanced Command Line for more information about why this works, but probably save that for later and finish this guide first.

Flags & Arguments

Most commands accept one or more arguments or flags. These are options that control how the command behaves.

Arguments

For example in the command

echo "foo"

The command is echo and it is given a single argument of "foo".

Flags

Flags are just special arguments, usually beginning with one or more dash - characters.

For example, the -n argument here is known as a flag or also sometimes called a parameter or a switch because the flag changes or “switches” the behavior of the command.

echo -n "foo"

Short Flags

Flags often come in short and long variations. Short variations are faster to type, but long variations are more descriptive.

For example, you may be able to run either of these variations

  • blarg --output-file=hello.txt this is the long variation
  • blarg -o hello.txt this is the short variation

These both do the same thing, but the long varition is more obvious as to what it does because the flag itself is descriptive.

I recommend always using long flags as you learn and in your shell scripts. Only use short flags when it is personally convenient to do so as you directly enter commands by hand.

Getting Help

Both Bash and PowerShell have common ways to find information about commands.

Bash

Most Bash commands respond to the --help option.

For example, to get some helpful information about the ls or list command in Bash, simply run the ls command and add --help like this.

elmo@doge:~$ ls --help

Alternatively you can also check the Manual Pages or Info Pages for a command with the man or info commands respectively.

elmo@doge:~$ man ls
elmo@doge:~$ info ls

PowerShell

Similar to Bash, PowerShell has standard ways to get help and documentation for commands.

Usually this is done with the -? flag, where ? represents that you have a question about the command, hence the question mark.

PS C:\> Get-ChildItem -?

Getting Around

You will want to be able to move around into different folders on your computer so you can more easily work with your projects and better manage your files.

Where Am I Now?

You can check which directory you’re in right now.

Bash

In Bash use pwd which stands for “print workding directory”.

pwd
/home/elmo

PowerShell

In PowerShell use Get-Location, the name is hopefully self explanatory.

Get-Location

Path
----
C:\Users\elmo

Changing Directory

Your current working directory can be changed.

Bash

In Bash use the cd command which stands for “change directory”.

elmo@doge:~$ cd projects
elmo@doge:~/projects$ pwd

PowerShell

In PowerShell use the Set-Location command, the name is hopefully self explanatory.

C:\Users\elmo> Set-Location projects
C:\Users\elmo\projects>

In both of these cases the prompt changes to show us that we are in the new different directory.

If your prompt doesn’t show this, use the technique above to see where you are right now.

Managing Your Files

Listing Files

Your can print out a list of files in the current directory

Bash

In Bash use the ls command which stands for “list”. Doing so should look something like this.

elmo@doge:~$ ls
Documents Downloads MyFile.txt MyPhoto.png
elmo@doge:~$

In Bash use ls -l for more details, think of -l as meaning “long”.

elmo@doge:~$ ls -l
total 112
drwxr-xr-x 2 elmo elmo 36864 Nov 28 2020 Documents
drwxr-xr-x 6 elmo elmo  4096 Nov 28 2020 Downloads
drwxr-xr-x 1 elmo elmo 20480 Nov 28 2020 MyFile.txt
drwxr-xr-x 1 elmo elmo 20480 Nov 28 2020 MyPhoto.png
elmo@doge:~$

PowerShell

In PowerShell use the Get-ChildItem command. Doing so should look something like this.

C:\Users\elmo> Get-ChildItem

    Directory:  C:\Users\elmo

Mode                LastWriteTime     Length Name
----                -------------     ------ ----
d----        06/28/2022   9:00 AM        1   Documents
d-r--        06/28/2022   9:00 AM        1   Downloads
-a---        06/28/2022   9:00 AM    54.88KB MyNotes.txt
-a---        06/28/2022   9:00 AM    72.33MB MyPhoto.png

Don’t worry too much for now about what all these details mean, you can deal with that when you want to.

Looking Inside Files

You can print the contents of a file.

Let’s say you have a file named Surprise.txt in your current folder.

Bash

In Bash, use the cat command to print it out.

elmo@doge:~$ cat ./Surprise.txt
Happy Birthday!

PowerShell

In PowerShell, use the Get-Content command to print it out.

PS C:\Users\elmos> Get-Content .\Surprise.txt
Happy Birthday!

Creating Directories

You can create new folders.

Bash

mkdir MyNewFolder

PowerShell

New-Item -Path MyNewFolder -ItemType Directory

Creating Files

You can create new files.

Bash

touch SecretRecipes.txt

PowerShell

New-Item -Path SecretRecipes.txt -ItemType File

Copying Files

You can copy files.

Bash

In Bash, cp stands for “copy”

cp SecretRecipes.txt NotSoSecret.txt

PowerShell

Copy-Item -Path SecretRecipes.txt -Destination NotSoSecret.txt

This will make a new file, the new file will be a copy of the old file and the old file will remain.

Moving & Renaming Files

You can move files. Moving a file is basically the same as renaming it.

Bash

In Bash, mv stands for “move”

mv ./locked/SecretRecipes.txt ./open/NotSoSecret.txt

PowerShell

In PowerShell

Move-Item -Path .\locked\SecretRecipes.txt -Destination .\open\NotSoSecret.txt

This will rename the old file. The file will remain but it will have a new name. No copy is made, only the one original file still exists.

This will also move the file from the locked folder into the open folder.

Deleting Files

You can delete files.

Bash

In Bash, rm stands for “remove”

rm ./some_file_i_dont_want.jpg

PowerShell

Remove-Item .\some_file_i_dont_want.jpg

You may notice it is much faster to use the command line to delete files than it is to use the Recyle or Trash bins.

Deleting Folders

You can delete folders, but if you want to remove an entire directory and everything it, you must do it recursively.

Bash

In Bash, -r means “recursive” and -f stands for “force”.

rm -r -f ./some_folder_i_dont_want

PowerShell

Remove-Item -Recurse -Force .\some_folder_i_dont_want

Running Programs

You can run a program by typing a fully qualified path to the program.

For example…

Bash

/usr/bin/uptime

PowerShell

C:\Windows\System32\notepad.exe

Running Relative Programs

You can also run a program using a relative path. You will probably want to cd into the program directory first.

For example

Bash

elmo@doge:~$ cd /usr/bin
elmo@doge:/usr/bin$ ./uptime

PowerShell

C:\> cd C:\Windows\System32
C:\Windows\System32> .\notepad.exe
