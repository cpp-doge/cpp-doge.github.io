What Is The Difference?

The difference from C to C++ is very great.

Most C++ code is not valid C code.

Most idiomatic C code is very poor C++ code.

Why Should I Use C++?

C++ has many features in both the language and its standard library including, in no particular order

  • Metaprogramming
  • Variadic Templates
  • Lambdas
  • Concurrency and Parellelism
  • Atomics
  • Structured Bindings
  • Regular Expressions
  • Strings, Arrays & Dictionaries
  • Unicode Support
  • Date & Time Functions
  • File And Directory Management
  • Random Number Generation
  • Function Overloads
  • Operator Overloads
  • Classes
  • Inheritance
  • Polymorphism
  • Iterators
  • Ranges
  • Concepts
  • Views & Spans
  • Exceptions
  • Variant

Plus there exist many other common useful libraries for things like

  • Network Programming
  • HTTP, REST & Websockets
  • Gregorian Calendar & DST Handling
  • GPU & Parallel Computation
  • Lexing, Parsing & Tokenization
  • Databases
  • Command Line Options
  • Unicode Conversion
  • String Formatting
  • 3D Graphics Math
  • Physics
  • Serialization
  • Unit Tests
  • JSON
  • Yaml

C has basically none of this. Or where C does have some of this, C has tremendously less of this and it is unsafe.

C++ can use any C library, but C cannot use any C++ library.

If the idea of rewriting all of this stuff from scratch is appealing to you, then you might like C. But you could also just rewrite it yourself in C++ instead, in a much safer and less painful way.

But!

But C++ Is Complex!

If you stick to a subset of modern idiomatic value semantics based C++ then life can be quite pleasant in C++ land.

A Paradox

C is simple

  • This means your C programs must be excessively verbose and complex as a result.

C++ can be complex

  • This means, however, that your C++ programs can be clean, concise, elgant, simple and safe as a result.

Much (if not most) of the C compiler source code for GCC has been rewritten in C++ because C++ is better, safer and more powerful.

But C Is Fast!

Not really. I mean, it can be, however…

C++ is not inherently slower than C.

In some cases C++ is faster than C precisely because C++ can better optimize due to a better type system. The canon example here is qsort.

C++ is only “pay for what you use”.

To do the same things in C would incur the same performance penalties, and your code would still be unsafe, less readable and more difficult to work with.

But! Part Deux

Why Shouldn’t I Use C?

C is an ancient outdated byte processing language invented in the pre-dawn of modern computing.

  • C has no features.
  • C is unsafe.
  • C is verbose and cumbersome.
  • If you can write it in C, you could write it in C++ instead.

C has basically none of the features of C++ like being able to easily and safely add things to a list or a dictionary.

C has no metaprogramming and no overloads.

If you write C then you will probably end up rewriting all these things yourself, poorly, with lots of bugs.

The C++ standard library and other well known libraries like Boost are written by experts. They have done it much better than you or I probably would.

But I Am Smart! I Can Handle It!

I believe you…

Until, one day, you forget to free some memory, because like the rest of us you are human.

Until, one day, you make a big mistake because the wall of text you are looking at would make Satan cry and you didn’t notice one of the thousands of small details that can shoot you in the foot, so now you are crying because prod is down and it’s 4PM on Friday.

Even if you can avoid this, which you probably can’t…

Work smarter not harder…

  • Why use your awesome brain power needlessly attending to pedantic details that would otherwise make your program unsafe?
  • Why use your awesome brain power to rewrite a bunch of stuff that a bunch of experts have already done for you, expertly?
  • Why use your awesome brain power to write a whole bunch of code you probably didn’t need to write?
  • Why make it harder for the next person?

Why Does Anyone Use C?

When your vendor provides a C compiler and not a C++ compiler, then you may have no choice.

When your vendor provides a good C compiler but a bad C++ compiler, then you may have little choice.

These situations are increasingly rare.

Are you stuck on an embedded platform, like a microcontroller for self driving cars, with 256 bytes of memory and no standard library? Maybe C works fine there, but C++ would work fine too if you have a C++ compiler available.

Most die hard C enthusiasts are probably just uninterested to try new things when what they already know gets the job done. It is their right to make this decision. We all probably want to retire some day and learning new things is hard.

The job could still usually be better and more safely done with C++.

If you have the choice, then choose C++.
