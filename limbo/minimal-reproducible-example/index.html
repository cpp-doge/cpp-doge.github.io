Basics

TLDR

Shorten your code and post it to Wandbox.org

See also: Howto Use Wandbox Video

What Is It?

A Minimal Reproducible Example is exactly enough code to demonstrate your problem, but no more and no less.

Sometimes also known as a Minimal Complete and Viable Example or MCVE.

How Do I Make One?

See: Howto Use Wandbox Video

Use an online compiler website that can compile and run your code like

  • Wandbox
  • Godbolt

For smaller examples you can also use the discord bot to compile and run your code.

I advise using an online paste site for your examples.

There is no syntax highlighting on Discord mobile and long lines are very difficult to read on mobile. Online compiler websites have syntax highlighting. Websites also usually allow pinch zooming for mobile users.

What Do I Show?

It Needs To Be Minimal

If you have too much code then start over from scratch.

Copy paste some of your old code if you like.

Strive to make your example fewer than about 50-100 lines.

See Also: But I Can’t Split It Up!

It Needs To Be Complete

We need all the details.

Maybe you made a typo we can’t see?

See also: Why Do I Need To Show Code

Describe The Problem

This is part of being complete.

Explain in regular English language what the problem is.

Posting code is good, but it’s only one of the first steps.

If you have a problem description, like from a leetcode challenge or a homework assignment, then post the full problem description.

Show the input and expected output. Explain how what you get is different from what you want.

It Doesn’t Work

Then post the full error message.

Simply saying “it doesn’t work” is generally insufficient.

There Is No Error Message

Then show the input and expected output. Explain how what you get is different from what you want.

I somewhat doubt there is no error, but either way do your best to describe how it is failing.

Saying something like “expected output is a full working video game” is not adequate and arguably a bit disingenuous. It is overly general, it is not specific, it does not help us help you. Of course everyone wants their code to create a working program, it would be extremely rare that a programmer wants their code to NOT result in a working program.

Saying something like “When I run it, it pauses and then immediately exits with no output”, while vague, is still better and more descriptive than saying “it doesn’t work”.

It Needs To Be BOTH Minimal AND Complete

We need to understand the full problem, but most of us don’t want to analyze thousands of lines.

Creating a good minimal example is an art and a delicate balancing act. It’s also a valuable skill to have.

Not too much, not too little.

It Should Compile And Run

The example you provide should compile and run.

Your example should include all your headers and your main() function.

It Should Be Nicely Formatted

You can easily format your code automatically on Discord.

You can also use automatic formatting tools like Clang Format and Clang Tidy.

I advise using an online paste site for your examples.

There is no syntax highlighting on Discord mobile and long lines are very difficult to read on mobile. Online compiler websites have syntax highlighting. Websites also usually allow pinch zooming for mobile users.

Why Should I Do This?

In the best case:

  • You will often discover and fix your own problems by doing this

In the worst case, if you still haven’t fixed it:

  • We are better and more quickly able to help
  • You are more likely to get more and better help
  • You are being kind to your fellow community members

Really Tho Why?

Ok, you’re still not convinced…

Why Do I Need To Show Code?

I shouldn’t really need to explain why we need to see code, but here goes.

Showing code is basically the bare minimum entry point for getting help on a programming help server.

For starters, doing so probably demonstrates that you are not lazy and you are not asking for someone to just hand you a solution, asking for which would be academic dishonesty. Most people are loathe to help the lazy and or the academically dishonest, not to mention that academic dishonesty is against the rules.

Code is precise and generally unambiguous. Without seeing at least some code we are left only to guess what your code might look like, what the problem might be, and what the best solution might be.

By showing code, your situation is much more immediately and abundantly clear. We can eliminate all kinds of potential ambiguities, miscommunications and infinite other blocking issues that could be inherent in your current endeavor and prevent you from getting help.

Showing code doesn’t necessarily illuminate your situation perfectly, but it’s generally the bare minimum place you need to start.

For example you could fail to show enough code, or you could make the mistake of showing too much, or your code could be just bad, poorly formatted, with bad variable names and difficult to understand. Plenty of other problems can still exist even when you show code, but showing code is still thousands of times better than not showing code.

We can’t advise how to fix or improve code we can’t see.

See also: It Needs To Be Complete

Defining The Problem

If you need to show more than 100 lines then you have not isolated and defined the problem very well.

If you can’t isolate and define the problem, then you don’t understand the problem; this is bad. It’s not reasonable to ask for help with something we don’t understand, much less something you don’t understand. In a case like this you may have Hidden Requirements that are not obvious, maybe that you weren’t even aware of.

Why Does It Need To Be Minimal?

Even a few lines of code is unintelligable when using Discord mobile. RIP anyone considering helping you on mobile when your code is even 50 lines long.

There is some wiggle room, 100 lines is a grey area. It is not a crime to post a 101 line example. Some people are willing to help with large examples. However, at some point too much is too much.

The more code you have, the more work you’ll need to do to ensure your example is relatively easy to work with. At least if you want a decent chance at getting good help.

Why Does It Need To Be Complete?

What is wrong with the following code?

void foo(qux q) {
  bar(q);
}

Who knows? We don’t know what qux is and we don’t know what bar is.

I believe that you may believe that you have provided all the necessary information, but maybe you have not?

It’s an easy mistake to make because you know your code but we do not.

Why Should It Compile And Run?

This way we can easily and quickly run it, see what’s wrong, make changes and give it back to you showing how to fix it, and you can see that it compiles and runs and that it is fixed.

If it doesn’t compile then we want the FULL ERROR MESSAGE and a complete BUT MINIMAL example that demonstrates the error, and it should be complete enough to compile and run when the error is fixed.

Why BOTH Minimal AND Complete?

We need enough info to help, but we also probably don’t want to read hundreds or thousands of lines. We do this for free.

If adding enough code for us to understand the problem means adding too much code then you probably have not done a very good job of isolating and describing the actual problem.

If the problem is not well isolated and well defined then it will be difficult to provide good help, or maybe any help at all.

Start over from scratch and try again until you have something small enough to be reasonable for a normal human but that still shows the full problem.

You may not always have the luxury of readily available free support. Being able to investigate and solve your own problems in a pragmatic, evidence based way is then the only option and is therefor a critical skill to have.

Why Should It Be Nicely Formatted?

First of all, it’s easy to do. You can use automatic formatting tools like Clang Format so there is no good reason not to do this.

Second. If it makes my eyes bleed, am I going to be motivated to help?

For Every Question?

Yes, most programming questions probably do warrant a minimal complete and viable example.

You should assume you need to post code.

Really Tho Every Question?

Not every question requires an example.

Like: “How many bytes are there in an int?”

This can be easily answered and does not depend on your other code.

Even if a minimal example is less warranted, it will always still be beneficial and productive and will more effectively communicate your question.

And you will always still get better help.

An Example

Bad

Question: “Why is this throwing an error?”

#include <string>

arr[20] = s;

cout << arr[20] << endl;

This is bad because there is a lot of code we cannot see and this code isn’t well formed anyway, this won’t compile as written.

What is s, what is arr, where is main(), is that all the headers? Is this code from different files? I have a thousand more questions.

Good

Question: “Why is this throwing an error?”

#include <iostream>
#include <string>

int main() {
  std::cout << "My Example" << endl;

  int arr[3] = { 1, 2, 3 };

  std::string s = "hello";

  arr[20] = s;

  cout << arr[20] << endl;
}

There are several problems with this code, can you spot them all?

This is good because now we can see a bunch of relevant details that help us understand where the problems are.

This is also good because someone can easily make a couple small changes to fix the code and send it back to you.

Then you have a full, thorough and complete solution to your problem because you provided a full, thorough and complete, yet minimal example.

Another Example

Bad

Question “I have six files, three .cpp and three .h files. I am supposed to write functions for a linked list.”

First of all, this isn’t a question, but let’s pretend the following code accompanies the post.

node.h

class Node {
public:
  int val;
  Node* next;
};

list.h

#include "node.h"

class List {
  Node* head;
  int size;

public:
  List();
  void append(int val);
  void insert(int val);
  void remove(Node* n);
  void pop();
  int length();
};

There are a several problems with this

  • This is not a specific question, nor really a question at all
  • We are not going to write it for you
  • Only two of the claimed six files are shown
  • It would seem you are implicitly asking someone to do it for you

This is better than nothing, but it’s still less than the bare minimum. Posting any code at all is only the first step. The code you post still needs to resemble a minimal viable example as described in this document.

The code in this example shows no implementation, so it’s near impossible to ask a specific question in the first place.

Please post more relevant code and ask a specific question. Even if you ask a specific question you need to show that you have at least tried to write some of these functions yourself.

But Mine Isn’t A Linked List

Being a linked list is not the point, the point is that you need to post enough code and ask a specific question, or at least any question at all.

Good

What would be good is to show all six files mentioned in the description.

What would be good is to ask any question at all, and better to ask a more specific question.

What would be good is to show that you have at least tried it yourself first.

The main point is that we won’t do it for you. This is an illustrative example. There are many reasons I won’t show a complete linked list implementation for you here.

Complaining Time

Why Do I Have To Do All This?

Strictly speaking, you don’t.

However…

  • You will get more and better help when your examples are good
  • People respect hard work, dedication and study
  • People are more likely to help you when you show that you have put in effort
  • If you show me that you care about programming, then I care to help you
  • People are providing help to you for free, usually in their spare time
  • Many of us have stressful full time programming jobs already
  • We probably don’t want to analyze hundreds or thousands of lines of code

But I Can’t Split It Up!

Yes you can.

But That’s More Work For Me!

Yes, it is.

But I Really Can’t Split It Up!

Yes, you probably can. This is probably just for lack of creativity on your part. Maybe also for lack of trying?

I claim literally any and every problem is in fact reducible to say about a hundred lines of code or so. Otherwise you haven’t well isolated and defined the actual problem.

See also Defining The Problem

Isolating and defining the problem well is important for getting good help, much less any help at all. If we can’t understand the problem (or if it takes too much effort to do so), we will be unable (or unwilling) to well advise on what the solution should look like.

Show what you are trying to achieve in isolation. Surely most of your code is not required to be shown to fully demonstrate the key issue at hand. Surely we also need enough code to understand the problem, hence needing to be at the same time minimal yet also complete.

But I Really REALLY Can’t Split It Up!

Yes you can. Get creative.

But That’s More Work For Me!

Yes, it is.

That’s WAY Too Much Work For Me!

Then that’s probably unreasonable to ask of someone else too, no?

Large Or Advanced Projects

The nature of online help forums is that they have limits. Online help forums are not fully suitable to help with large or advanced projects.

They are here to provide general guidance, facilitate intuition, help you with common problems, soften sharp corners, illuminate best practices, and provide further resources to supplement your study.

See Also: Your Job

SFML, SDL2, GLFW, etc

Sadly this is true. However…

At some point when you get past beginner phase, you need to learn to solve your own problems.

See Also: Your Job

If you still need help you then do your best to make a minimal reproducible example anyway, following the instructions in this guide.

It’s ok if it doesn’t compile and run in a case like this but we still need enough information, and not too much information.

See Also: A Last Resort

Your Job

At some point in your programming career you need to learn to solve your own problems. At some point, when you get a lot of code and or code that is well beyond beginner level (like a 2D/3D game), then dealing with it becomes your job, and we probably don’t want to deal with what likely has been a full time job worth of effort on your part.

I’m sorry that this is the case, but this is the case. At some point, to deal with this much code, you usually need to pay someone. This is part of why we get paid at our own jobs in the first place.

If no one is helping then consider doing your best to create a minimal complete and viable example anyway.

A Last Resort

As a last resort you could try just putting all your code in a GitHub repo and giving us the link. Will someone want to read that much code? Probably not, which is why you should still probably make an MCVE, but maybe someone will, idk.

Good luck.
