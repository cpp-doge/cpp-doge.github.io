Motivations

In the beginning, C++ YouTube tutorials for newbies commonly recommend using VSCode, often in conjunction with MinGW for Windows.

This has made a lot of beginners very angry and has been widely regarded as a bad move.

It is extremely common that beginners experience frustration because using these generally requires extensive experience and skill. Most beginners do not have this yet.

Using an IDE like Visual Studio means you can avoid the need to study a bunch of what is, for a beginner, probably irrelevant information, and instead focus on learning C++.

VSCode is not a build system so there often arises confusion around this fact and needing to configure both the build and the syntax checking for the editor. Many other similar issues also arise for most beginners when trying to use VSCode.

Visual Studio is an all in one programming tool with a GUI. You can configure your project by simply right clicking and browsing through dialog boxes and menus etc. No need to memorize command line compile flags or other details before you can get to work learning C++ and writing programs. Visual Studio is both a code editor and a build system (and much much more), when you have configured your project once, you are basically done and you can get to programming.

VSCode is good, but I don’t recommend it for beginners or even up to intermediate level.

See also Why Not MinGW

Why An IDE

An Integrated Development Environment or IDE is a programmer’s tool that usually comes with most if not all the features you need to get up and running quickly writing your own programs.

  • You don’t really need to know what you’re doing yet
  • It is nearly zero effort and generally pain free
  • An IDE usually has tons of features including but not limited to
    • Compiler
    • Build System
    • Debugger
    • Terminal
    • Error Checking
    • Auto Code Completion
    • Code Tooltips
    • Jump To Definition
    • Jump To Include File
    • Search In All Files
    • Package Manager
  • Visual Studio at least has all of this, other IDEs may vary
  • You need to know what you’re doing already to use VSCode effectively
  • VSCode will almost certainly be painful for beginners

Using an IDE is arguably still somewhat complex, but you can do most of what you need through graphical menus, buttons, dialog boxes etc so you don’t need to memorize low level details like compiler specific command line flags etc.

Why Visual Studio

Visual Studio is an IDE.

Visual Studio is widely considered very good, maybe even the best IDE of all IDEs on any OS.

Visual Studio is the easiest, simplest and fastest way to get into C++ programming on Windows.

Visual Studio is basically point, click and run. No fuss. It does a ton of things for you so you can focus on writing and learning C++.

Visual Studio comes with a compiler and a build system, you don’t need to install other tools like MinGW or GNU Make.

Configuring Visual Studio requires some effort but doing so is mostly trivial to configuring other tools like VSCode.

There is less confusion and less work to do because Visual Studio already comes with a compiler and a build system.

CMake can generate Visual Studio projects.

Download: Visual Studio

If you are not on Windows you will need to use Something Else as a C++ IDE because Visual Studio for C++ is not available on Mac or Linux.

What Even Are These Things

Does Doggo A Bamboozle

Visual Studio is NOT THE SAME as VSCode despite both of them having similar names.

The Microsoft web pages for downloading these products also seem to want people to believe these products are the same, but they are not.

What Is Visual Studio

Visual Studio has everything you need to get up and running quickly writing C++ programs with very little effort.

Visual Studio is a FREE fully featured Integrated Development Environment or IDE.

An IDE provides all the features you need to get up an running quickly writing programs. It includes a compiler, a build system, a debugger, sanitizers, error checking, code completion and more.

You can use Visual Studio for C++, Python, JavaScript, C#.NET and more.

What Is VSCode

Visual Studio Code (aka VSCode) is a programmer’s plain text editor useful for a wide variety of programming languages including JavaScript, Python C++ and almost any other language you can imagine.

VSCode is like a “Build Your Own IDE” kit. It will not likely work well out of the box without some configuration, though some plugins are better than others.

VSCode is quite good, however, using VSCode requires some skill and experience, or at least a willingness to put in some extra time and study.

What Is GCC

GCC is The GNU Compiler Collection, a compiler that originated from GNU/Linux.

GCC was originally (and still mostly is) intended for Linux.

What Is MinGW

MinGW is a just a special version of GCC that lets you write Windows programs.

MinGW means Minimalist GNU For Windows.

Why Not MinGW

If you don’t know what MinGW is and why you want it (other than a tutorial said so) then you should not use MinGW.

If you just want GCC for some reason then use GCC from the Windows Subsystem For Linux aka WSL.

See the documentation for Installing WSL.

I still don’t recommend WSL either, for the same reasons. Like VSCode, using WSL requires background knowledge and skill that most beginners don’t have and don’t need.

Reaons Not To Use MinGW

  • Visual Studio is also free and much better and easier for beginners
  • MinGW has a history of being incomplete or even bad
  • For example: MinGW was lacking a random device for a very long time
  • MinGW will probably always have less than 100% parity with native Windows
  • MinGW systems typically use Bash which may cause confusion and pain when dealing with Linux vs Windows differences like file paths and more
  • MinGW receives updates, but why use MinGW when it’s incomplete, sometimes bad, and VS is better and also free?
  • MinGW LLVM is not an appropriate solution either, especially when you could just use native Clang For Windows instead

Why Do People Use MinGW

Often because of bad YouTube tutorials.

Maybe also sometimes because of the GPL.

You probably don’t need to worry about the GPL.
