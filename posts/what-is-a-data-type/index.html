Before You Begin

See: How Binary Works

Computer Memory

Computers store information in memory as binary numbers.

Transistors

Transistors are good at turning on and off. This can easily be used represent a one or a zero.

The information in your computer is managed by a bunch of transitors. You can think of it like being one bit per transistor, meaning each transistor can represent either a zero or a one. It’s usually more complicated, but you can think of it like this.

Transistors can then be combined into larger binary numbers the same way we combine regular human digits into larger base ten numbers.

Bytes

Your computer hardware organizes your computer memory into small chunks called bytes. A byte is just the smallest group of transistors that can be chewed on at the same time, pun intended.

Your computer memory is layed out in a flat linear sequence of bytes. You can do stuff with more than one byte at a time, but not with less than one byte at a time.

A byte is 8 bits on most normal computers, iPhones Android Phones, Smart TVs etc, but is sometimes different in other places. Arduino uses 12 bits per byte.

You can probably assume that bytes are always 8 bits.

How Much Can We Count

People use tools like this “Tally Counter” to count things, maybe like people passing through a ticket gate to a football game.

This tally counter only has four digits, so how many things can we count until we run out of digits?

Well, we can count 9999 things before the digits reset back to zero.

Memory in a computer has a similar limitation. We can count as many things as we have binary digits, and then the number will reset or “roll over” to zero.

How Much Memory Can We Use

Memory in a computer is limited. Many people have a lot of RAM in their computers these days, but wouldn’t most of us still want more?

We sometimes place extraordinary demands on computers with video games, artificial intelligence, websites used by billions of people, video editing, photo editing, music production, 3D modeling and more.

How much memory do we need to store a single number? Why use more memory than we really need to for only a single number?

The answer is that these things have mostly already been chosen for you. You generally have the choice between four bytes per number or eight bytes per number. This corresponds to 32-bits per number and 64-bits per number. You have likely already heard these numbers talked about elsewhere.

Notice that, as with the tally counter above, this means we have a limited number of digits so we can only count a limited number things.

How Big Are We Talking

How big of a number can I store with 32-bits

  • Up to about four billion or 4,000,000,000
  • To be precise we can count up to 4,294,967,295
  • After that the number will roll back over to zero, like the tally counter

How big of a number can I store with 64-bits

  • Up to about eighteen quintillion or 18,000,000,000,000,000,000
  • To be precise we can count up to 18,446,744,073,709,551,615
  • After that the number will roll back over to zero, like the tally counter

Those Are Some Weird Numbers

Yep.

Digits in a computer are binary, so how many things we can count is different than if those digits were our familiar human base ten system.

What Should I Use

Ask yourself

  • Do I need more than a few quintillion things in my computer program?
  • If not then an eight byte 64-bit number is probably fine

Then ask yourself

  • Do I need more than a few billion things in my computer program?
  • If not then a four byte 32-bit number is probably fine

A billion things is probably more than enough for most people on most days.

Floating Point Numbers

Real numbers are numbers that have a fractional or decimal part. So far we have only talked about integers. A key important difference that we now come to is how we use the computer’s memory and what that memory means.

Real numbers can be stored using 32-bits each or 64-bits each. As with integers, either of the these is likely far more memory per number than most people will ever need.

There is a VERY IMPORTANT DIFFERENCE however, which is the binary digits mean DIFFERENT THINGS for a real number inside a computer than they do for an integer inside a computer.

Floating point numbers are too much to cover here, but the short simple answer is that we take a chunk of memory, let’s say four bytes, and then we split that conceptually so that some of the digits represent the number and some of those digits represent where the decimal point is.

Let’s invent a hypothetical 32-bit binary number for example purposes.

01000000010010001111010111000011

Let’s conceptually view this as four bytes of data because that’s how it lives in the computer.

01000000 01001000 11110101 11000011

If we treat this number as a floating point number instead of an integer then we might look at it like this

     This is the number part
                |
                |                 This is the decimal place
                |                           |
     -----------|------------               |
    /                        \           ---|--
   /                          \         /      \
  /                            \       /        \
 /                              \     /          \
|                                |   |            |
|                                |   |            |

   01000000  01001000  11110101         11000011

Exactly what this looks like may vary from computer to computer.

When we treat our binary number like this then this number happens to be 3.14 or approximately Pi in my computer.

However, if we treat it like a regular integer then it’s actually 1,078,523,331! That’s a big difference between about three and about a billion!

Plain Text

Well, we can store numbers in the computer. That’s good, we need numbers for things like video games and online shopping.

What about regular normal ish human language?

Well, the computer only stores numbers, binary numbers. How can we store some text?

The answer is to pretend that each number in the computer represents a letter of the alphabet.

This is where we get ASCII. ASCII is an official standard that associates numbers to letters.

Confusingly, the digits we normally use, themselves are mapped to other numbers in ASCII and those other numbers are different than the digits themselves!

Take a look a which numbers are mapped to which letters in the ASCII Table.

For example, in ASCII, the number 65 represents the capital letter A, the first letter of the alphabet. The number 51 actually represents the numeral digit 3! What!? Why isn’t 3 actually 3!?

That’s just how it is, so you pretty much just get to live with it for now, sorry.

A Plain Text Example

Let’s say we have five bytes of memory, and in those bytes we have the binary number.

10010001100101110110011011001101111

Let’s conceptualize this as five separate bytes

1001000 1100101 1101100 1101100 1101111

So what do these bytes mean? Well if we treat them as ASCII text then each byte is a single number that represents an ASCII character from the ASCII Table.

Let’s figure out what these numbers mean.

  • 1001000 in binary = 72 in base ten
  • 1100101 in binary = 101 in base ten
  • 1101100 in binary = 108 in base ten
  • 1101100 in binary = 108 in base ten a second time
  • 1101111 in binary = 111 in base ten

If we look up these numbers in the ASCII Table we find that

  • 72 = 'H'
  • 101 = 'e'
  • 108 = 'l'
  • 108 = 'l' a second time
  • 111 = 'o'

So these bytes, when treated as ASCII text mean Hello.

However, if we treat these bytes as an integer number instead we get the number 19,540,948,591! I dare say that the word Hello and the number about ninteen billion are very different things!

What Is A Data Type

Ok, so let’s review

  • A byte is the smallest chunk of memory we can use
  • We can use four byte numbers or eight byte numbers
  • We can pretend some bytes are text instead of numbers
  • How we view the bytes completely changes what the data is!

This is where we find a real need to start talking about the concept of a Data Type.

No Really What Is A Data Type

A Data Type, simply put, describes what is in some memory. Is it a number, if so what kind of number? Is it some text? If so, what kind of text, is it ASCII? Is it Unicode? If so, which kind of Unicode?

For this reason, we have Data Types. A data type basically just says what is in some memory, and maybe also says something about how much memory is used.

Some Simple Examples

For example in C++

  • The int data type means a 32-bit integer, one that can be negative
  • The long data type means a 64-bit integer, one that can be negative
  • The unsigned int data type means a 32-bit integer, one that can ONLY be positive
  • The unsigned long data type means a 64-bit integer, one that can ONLY be positive
  • The float data type means a 32-bit decimal number
  • The double data type means a 64-bit decimal number, or double the size of a float, hence the name

This may not always be perfectly true, but let’s roll with it for now.

Text Data Types

In the world of regular human language, like some English language text, we have the char data type which is a single byte of memory that represents some alphabet character.

We can store as many chars in a row as we want or need, we are not limited to four byte or eight byte chunks. We could store the entire works of Shakespear this way.

Your Own Data Types

C++ lets you create your own custom data types by writing classes.

We can also use a bunch of the classes that are already written for us that come from the C++ standard library.

These classes let us do things like easily deal with text, do advanced math like with complex numbers, read and write files, take user input, print stuff to the screen, check the current date and time and much more!

These classes, are themselves, data types! WOW!

Why Write Your Own

Why would you want to write your own data type? That’s a good question.

Some data you have might consist of a large chunk of memory like a bunch of integers or real numbers that form something bigger when considered as a whole. You might also have some text and numbers that belong together.

Examples might be

  • A vacation photo
  • A wedding video
  • A wheel on a self driving car
  • A 3D video game model
  • Some music like a song
  • Some audio like a podcast
  • A candlestick price on the stock market
  • A planet in a physics simulation
  • A molecule in a chemistry simulation
  • A database connection
  • A network packet
